# Contributing to BayOrder

First off, thank you for considering contributing to BayOrder! We welcome any form of contribution, from reporting bugs to submitting new features.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue and provide the following:
* A clear and descriptive title.
* A detailed description of the problem, including steps to reproduce it.
* Which part of the project is affected (e.g., "Customer App", "Owner Dashboard").
* Any error messages or screenshots.

### Suggesting Enhancements

If you have an idea for a new feature or an improvement:
1.  Open an issue and use the "feature request" label.
2.  Describe the feature in detail: what it does and why it's useful.

### Submitting Pull Requests

1.  Fork the repository to your own GitHub account.
2.  Create a new branch for your changes (e.g., `git checkout -b feat/add-new-payment-method`).
3.  Make your changes.
4.  Commit your changes with a clear commit message.
5.  Push your branch to your fork.
6.  Open a Pull Request (PR) from your branch to the main `BayOrder` branch.
7.  In the PR description, explain the changes you've made and link any relevant issues.

We will review your PR as soon as possible. Thank you for your contribution!
